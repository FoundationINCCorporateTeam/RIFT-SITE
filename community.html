RFC (Request for Comments) Process

For substantial changes, we use a formal RFC process to ensure community input and thorough design review.

  1. Proposal: Submit RFC document via GitHub PR
  2. Discussion: Community feedback period (minimum 2 weeks)
  3. Refinement: Author addresses feedback and updates proposal
  4. Decision: Core team votes to accept, reject, or defer
  5. Implementation: If accepted, implementation PRs can proceed

Release Philosophy

Semantic Versioning

We follow strict semver. Minor versions maintain backward compatibility. Breaking changes only in major versions with clear migration paths.

Stability Guarantees

v1.x series commits to language stability. New features are additive. Deprecations announced one minor version in advance.

Code of Conduct

Our commitment to an inclusive and respectful community

Core Principles

Respect & Professionalism

Treat all community members with respect. Focus on technical merit. Disagreements are natural; approach them constructively and professionally.

Inclusive Environment

Welcome contributors of all backgrounds and experience levels. Provide constructive feedback. Help others learn and grow.

Expected Behavior

  • Assume good intent: Presume positive intentions unless proven otherwise
  • Be clear and constructive: Provide specific, actionable feedback on technical matters
  • Accept responsibility: Own your mistakes and learn from them
  • Focus on the work: Critique ideas and code, not people
  • Support newcomers: Remember that everyone started somewhere

Unacceptable Behavior

  • Harassment, discrimination, or exclusionary behavior
  • Personal attacks, insults, or inflammatory comments
  • Trolling, intentional disruption, or sustained off-topic discussion
  • Publishing others' private information without consent
  • Other conduct that undermines a professional environment

Enforcement

Core maintainers are responsible for enforcing this code of conduct. Violations should be reported via GitHub or direct contact with maintainers.

Consequences: Depending on severity, actions may include warnings, temporary bans, or permanent removal from the project. All enforcement actions are documented and reviewable.
